# คู่มือติดตั้งระบบแจ้งเตือนลูกค้าผ่าน LINE (รายสาขา)

ระบบส่งข้อความอัตโนมัติผ่าน LINE OA ของแต่ละสาขา:

**เช่ารายวัน (แจ้งเตือนอย่างเดียว ไม่มี QR — ให้ลูกค้าแจ้งพนักงาน):**
- เตือนก่อนครบกำหนด 1 ชั่วโมง — "ใกล้ถึงเวลาคืน ถ้าต่อรบกวนแจ้งพนักงาน"
- ทวงเมื่อเกินกำหนด 3 ชั่วโมง (ถ้ายังไม่มีการกดต่อเวลาในแอพ)
- เกินครบ 1 วัน, 2 วัน, 3 วัน... ทวงซ้ำทุกวัน **พร้อมยอดค้าง** (จำนวนวัน × ค่าเช่า/วัน)

**เช่ารายเดือน (มี QR PromptPay + ยอดเงิน):**
- ถึงวันครบกำหนดจ่าย (payment day) — "ต่อสัญญาโอนตาม QR ส่งสลิปในแชท / ไม่ต่อนำรถมาคืนในเวลาทำการ"
- งานรูทีนถึงกำหนด (เช่น เปลี่ยนน้ำมันเครื่อง) — แจ้งลูกค้ารายเดือนที่ถือรถให้นัดเข้าเซอร์วิส

**แจ้งเจ้าของ — สรุปงานค้างวันละ 1 ข้อความ (หลัง 9 โมงเช้า) ส่งซ้ำทุกวันจนกว่าจะเคลียร์:**
- 📄 ภาษี / พ.ร.บ. ใกล้หมดอายุหรือหมดแล้ว ที่ยังไม่ได้ต่อ (ล่วงหน้า 30 วัน ตาม doc_alert_days)
- 🔧 งานเซอร์วิสถึงกำหนดที่ยังไม่ได้ทำ — น้ำมันเครื่อง/เฟืองท้าย ฯลฯ ทั้งครบตามวันและครบตามกิโล
- staff ทำรายการในแอพเมื่อไหร่ รายการนั้นหายจากสรุปวันถัดไปเอง

---

## ⭐ โหมดประหยัด: เริ่มแค่แจ้งเตือนเจ้าของก่อน (ยังไม่เปิดฝั่งลูกค้า)

ทำแค่นี้พอ — **ข้ามเรื่อง LIFF / PromptPay / ผูกไลน์ลูกค้าไปก่อนได้เลย:**
1. รัน SQL (ข้อ 1 ด้านล่าง)
2. ตั้ง CRON_SECRET บน Vercel + ตั้งเวลา cron ใน Supabase (ข้อ 3-4 ด้านล่าง)
3. เอา Channel access token ของ OA สาขาไหนก็ได้ (เช่น สาขาหลัก) วางช่อง **"LINE Channel Access Token"** ในหน้าตั้งค่า
4. หา User ID ตัวเอง: LINE Developers Console → channel นั้น → แท็บ **Basic settings** → ล่างสุด **"Your user ID"** (ขึ้นต้นด้วย U) → วางช่อง **"LINE Group / User ID"** — ไม่ต้องสแกนอะไรทั้งนั้น
5. มือถือนาย: แอด OA ตัวเองเป็นเพื่อน (ถ้ายังไม่เคย) — ไม่แอดระบบส่งหาไม่ได้
6. เปิดสวิตช์ "📄 เอกสารรถใกล้หมดอายุ" + "🔧 งานเซอร์วิสถึงกำหนด" → บันทึก

ใช้โควต้าแค่ ~30 ข้อความ/เดือน (วันละ 1 สรุป เฉพาะวันที่มีงานค้าง) — ฟรีสบายๆ
ฝั่งลูกค้าจะเปิดเมื่อไหร่ค่อยกรอก token รายสาขา (Section รายสาขาในหน้าตั้งค่า) — ไม่กรอก = ไม่ส่ง ไม่เสียเงิน

การกดในแอพมีผลทันที: **คืนรถ/จบสัญญา = หยุดแจ้งเตือนอัตโนมัติ**, **ต่อเวลา = เริ่มนับรอบใหม่จากวันครบกำหนดใหม่**
ข้อความแบบรายวัน (รายเดือน/รูทีน/เอกสาร) ส่งช่วงหลัง 9 โมงเช้า

## ขั้นตอนติดตั้ง (ทำครั้งเดียว)

### 1. รัน SQL ใน Supabase
เปิด Supabase → SQL Editor → รันไฟล์ `supabase-migration-line-notify.sql` (ส่วนบน ข้อ 1-3)

### 2. ตั้งค่า LINE Developers Console (ทำทีละสาขา)
เข้า https://developers.line.biz → เลือก Provider ของสาขานั้น

**2.1 Channel access token:**
- เปิด Messaging API channel ของ OA สาขานั้น (ถ้ายังไม่มี ให้เปิดใช้ Messaging API จาก OA Manager)
- แท็บ **Messaging API** → Channel access token (long-lived) → **Issue** → copy ไว้

**2.2 LIFF app (สำหรับผูกไลน์ลูกค้า):**
- สร้าง **LINE Login channel** ใน Provider เดียวกัน (ถ้ายังไม่มี)
- แท็บ **LIFF** → **Add** →
  - Size: `Tall`
  - Endpoint URL: `https://YOUR-DOMAIN.vercel.app/line/link`
  - Scope: ติ๊ก `profile`
  - Bot link feature: `On (Aggressive)` ← สำคัญ! ทำให้ลูกค้าแอด OA อัตโนมัติตอนสแกน
- copy **LIFF ID** ไว้ (หน้าตาแบบ `1234567890-AbcdEfgh`)

> ทำซ้ำข้อ 2 ให้ครบทุกสาขา (3 สาขา = 3 token + 3 LIFF ID)
> ถ้าสาขาไหนใช้ OA เดียวกัน ก็ใส่ token/LIFF ID เดียวกันได้

### 3. ตั้ง CRON_SECRET บน Vercel
- Vercel → Project → Settings → Environment Variables
- เพิ่ม `CRON_SECRET` = รหัสสุ่มยาวๆ (ตั้งเองอะไรก็ได้ เช่น สุ่มจาก https://randomkeygen.com)
- **Redeploy** 1 ครั้งให้ env มีผล

### 4. ตั้งเวลาเรียก cron ใน Supabase
- เปิดไฟล์ `supabase-migration-line-notify.sql` ส่วนข้อ 4 (ที่ comment ไว้)
- แก้ `YOUR-DOMAIN.vercel.app` และ `YOUR-CRON-SECRET` ให้ตรงของจริง
- เอา `--` ออก แล้วรันใน SQL Editor

### 5. กรอกตั้งค่าในแอพ
- เข้าแอพ → เมนูเจ้าของ → ตั้งค่า → **💬 LINE แจ้งเตือนลูกค้า (รายสาขา)**
- กรอกทีละสาขา: Channel Access Token / LIFF ID / เบอร์ PromptPay → บันทึก

## วิธีใช้งาน (staff ทำทุกครั้งที่ปล่อยเช่า)

1. สแกน QR รถ → เมนูพนักงาน → กด **💬 ผูกไลน์ลูกค้า**
2. ให้ลูกค้าสแกน QR บนจอด้วยมือถือ → เด้งเข้า LINE → กดยืนยัน
3. หน้าจอ staff ขึ้น ✅ ผูกไลน์เรียบร้อย = เสร็จ ที่เหลือระบบทำเองหมด

ลูกค้าที่ไม่มีไลน์/ไม่สแกน ระบบจะข้ามไป (staff ตามเองเหมือนเดิม)

---

## ทดสอบระบบ

- ทดสอบ QR PromptPay: เปิด `https://YOUR-DOMAIN.vercel.app/api/line/qr?branch=<branch_id>` ในเบราว์เซอร์ ต้องเห็นรูป QR
- ทดสอบ cron: เปิด `https://YOUR-DOMAIN.vercel.app/api/cron/line-notify?secret=<CRON_SECRET>` ต้องได้ `{"checked":N,"sent":N,"failed":0}`
- ทดสอบจริง: สร้างสัญญาเช่าที่ครบกำหนดในอีก ~50 นาที ผูกไลน์ แล้วรอ cron รอบถัดไป (ไม่เกิน 10 นาที) ต้องมีข้อความเข้าไลน์

## ค่าใช้จ่ายที่เกี่ยวข้อง

- LINE OA แผนฟรี: push ได้ ~300 ข้อความ/เดือน/OA (1 การแจ้งเตือน = 2 ข้อความ เพราะมีรูป QR)
- ถ้าลูกค้าเยอะ อัพเกรดแผนของ OA สาขานั้นตามจำนวนที่ใช้จริง
